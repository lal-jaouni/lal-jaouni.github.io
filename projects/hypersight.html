HyperSight

A full-stack SaaS analytics platform built from scratch in 12 weeks. Natural language queries, RAG-enhanced AI insights, and support for 100+ LLM providers — designed to make data accessible to non-technical users.

Role Sole Developer & Architect
Timeline 12 weeks, 13 MVPs across 9 phases
Scale 64,300+ lines of code
Stack FastAPI, React, PostgreSQL, Docker
AI LiteLLM, pgvector, RAG, NL2SQL
Screenshots coming soon

The Problem

Most analytics platforms assume technical users. They require SQL knowledge, familiarity with data pipelines, and comfort navigating complex dashboards. For organizations where the people making decisions aren't the same people writing queries, that's a wall.

HyperSight was designed to remove that wall — let anyone ask questions of their data in plain English and get real, trustworthy answers backed by the actual database.

What I Built

A complete enterprise platform covering data ingestion, transformation, analytics, AI-powered insights, and collaboration — all built solo across 13 MVPs in 9 development phases.

Natural Language to SQL

Users type questions like "What were our top 10 products last quarter?" and the system translates that into validated SQL, executes it against their warehouse, and returns formatted results. The NL2SQL pipeline includes schema introspection, query validation, and result caching.

RAG Knowledge Base

Built a retrieval-augmented generation system using pgvector embeddings. Users can index internal documents (63+ in production), and the AI chat interface draws on both the knowledge base and live database context to answer questions with citations.

Multi-Provider LLM Support

Integrated 100+ LLM providers through LiteLLM, giving organizations flexibility to use OpenAI, Anthropic, local models, or anything in between. Provider switching is seamless and configurable per-user.

Full Analytics Suite

Beyond AI features, the platform includes dashboards with KPI tracking, exploratory data analysis, time-series forecasting (Facebook Prophet), A/B testing with statistical rigor, collaboration tools, and a data dictionary for institutional knowledge.

Architecture

The backend runs on FastAPI with async SQLAlchemy, serving 100+ REST endpoints across 24 routers. The frontend is TypeScript React with strict mode, 112 components, and a 59% optimized bundle (108 chunks down to 44, under 220 KB). Six Docker containers orchestrate the full stack.

Backend

Python 3.11+, FastAPI, SQLAlchemy (async), Pydantic, Alembic, 100+ endpoints, 28 service classes, 370+ tests

Frontend

TypeScript 5.3 (strict), React 18, Vite 5.0, Material-UI v5, 112 components, Zustand, TanStack Query v5, AG Grid

AI / ML

LiteLLM (100+ providers), pgvector, sentence-transformers, RAG pipeline, NL2SQL, Facebook Prophet, SciPy

Infrastructure

PostgreSQL 15, Docker (6 containers), GitHub Actions CI/CD, Sentry error tracking, dbt transformations

What This Taught Me

Building an entire SaaS platform solo in 12 weeks forced clarity on what matters at each stage. I learned to make architectural decisions that would survive rapid iteration — choosing async-first patterns, keeping services loosely coupled, and investing early in testing (370+ tests, >80% coverage) so that later phases didn't break earlier ones. The AI integration work also deepened my understanding of how to make LLM-powered features reliable enough for production use.
