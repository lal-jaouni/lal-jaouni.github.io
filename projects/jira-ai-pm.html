Jira AI PM

An AI-powered project management tool that turns meeting transcripts into structured Jira tickets. Paste in your meeting notes, and the system extracts action items, detects duplicates semantically, and creates tickets with human-in-the-loop approval.

Role Sole Developer
Scale 10,000+ lines of code
Stack FastAPI, React, PostgreSQL, Celery
AI Multi-LLM, sentence-transformers, pgvector
Key Result 70-80% less manual task creation

The Problem

After every meeting, someone has to manually translate discussion into action items, check whether those items already exist somewhere in Jira, write up tickets with the right context, and assign them. It's tedious, error-prone, and leads to duplicate work when the same task gets created from different meetings.

What I Built

Transcript Processing

Upload a meeting transcript (or paste raw notes), and the system uses an LLM to extract structured action items — task descriptions, priorities, assignee suggestions, and relevant context. The extraction prompt is tuned to pull out commitments and next steps, not just topics discussed.

Semantic Duplicate Detection

Before creating any ticket, the system checks for semantic similarity against existing Jira issues. Using 384-dimensional embeddings (all-MiniLM-L6-v2) stored in pgvector, it computes cosine similarity to surface potential duplicates — even when they're worded completely differently. This cut duplicate tickets by roughly 50%.

Multi-LLM Support

Supports three LLM providers out of the box: Ollama (for local/private use), OpenAI, and Anthropic. Switching between them is a configuration change, so teams can choose based on their privacy requirements and budget.

Human-in-the-Loop Workflow

Nothing goes to Jira without approval. Extracted tasks are presented for review — edit descriptions, adjust priorities, dismiss false positives, resolve duplicates — then batch-create with one click. The system handles the Jira API integration, credential encryption, and ticket formatting.

Async Processing

Long transcripts and LLM calls are processed asynchronously through Celery with Redis as the message broker. The UI stays responsive while tasks are being extracted and embeddings computed in the background.

Why This Matters

The most useful AI features aren't the ones that automate everything -- they handle the tedious parts while keeping humans in control of decisions. The duplicate detection alone saves more time than the task extraction, because catching a duplicate before it's created prevents confusion downstream. Trust in AI systems comes from transparency, not accuracy alone -- the same principle behind the safety guardrails in clinical RAG systems.
